## 📦 커밋 컨벤션

> ✅ 모든 커밋은 아래 규칙을 따릅니다.

### 📌 커밋 태그 규칙

| 플랫폼    | 태그    |
|--------|-------|
| 백준     | `BOJ` |
| 프로그래머스 | `PGS` |

### 📌 커밋 메시지 형식

```bash
[플랫폼] 문제번호_문제이름
```

### 📌 일반 작업 커밋 태그

| 태그         | 설명                           |
|------------|------------------------------|
| `feat`     | ✨ 새로운 기능 추가                  |
| `fix`      | 🐛 버그 수정                     |
| `refactor` | 🔨 코드 리팩토링 (기능 변화 없음)        |
| `comment`  | 💬 주석 추가 또는 오타 수정 (동작 변경 없음) |
| `docs`     | 📚 문서 수정 (README 등)          |
| `merge`    | 🔀 브랜치 병합                    |
| `rename`   | 🚚 파일/폴더명 수정 또는 이동           |
